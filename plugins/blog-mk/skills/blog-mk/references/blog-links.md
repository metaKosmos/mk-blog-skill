## REGRAS DE LINKAGEM (Atualizado v4.1)

### Quantidade mínima
- **Mínimo 6-8 links internos** por artigo (era 4 — subiu na v4.1)
- 1-2 CTAs com UTM apontando para `https://form.respondi.app/L4NmIy24`

### Composição obrigatória
1. **1 Pillar Page** do mesmo pilar do artigo
2. **2 cross-pilar** (artigos de outros pilares relacionados)
3. **2 mKases** (clientes citados → linkar para `/mkases/[slug]/`)
4. **1-2 LPs de produto** (soluções mK citadas → linkar para LP)
5. **1-2 CTAs** apontando para `https://form.respondi.app/L4NmIy24`

### Regra "first mention link"
- Marca cliente citada pela 1ª vez → linkar nome para `/mkases/[slug]/`
- Solução mK citada pela 1ª vez → linkar nome para LP (`/mk-3d-shop/`, `/mk-beauty/`, etc.)
- **Não repetir** o mesmo link 2 vezes no mesmo artigo (escolha o melhor parágrafo para o link)

### Regras gerais
- Inserir no fluxo do texto com contexto: "Para entender como [X] funciona, o artigo sobre [título] explica..."
- NÃO criar seção separada "Leia também" — integrar naturalmente
- Priorizar artigos do mesmo pilar ou pilares adjacentes
- **JAMAIS linkar concorrentes**
- URLs verificadas via `sitemap-urls.md` — posts ficam na RAIZ do site (sem `/blog/`); mKases em `/mkases/[slug]/`
- UTMs com `&amp;` encoding em atributos href
